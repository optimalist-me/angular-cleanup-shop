# Angular Governance Program

> We reduce cost of change in Angular codebases by restoring architectural discipline — without rewriting your app.

Angular Governance Program is a governance-first stabilization program for Angular and Nx codebases that outgrew informal structure.

## Core Positioning

This is not a maintenance subscription and not a feature delivery service.

This is a narrow governance engagement focused on:
- structural discipline
- measurable cost-of-change reduction
- safer upgrades
- smaller, safer pull requests
- clearer ownership boundaries

## For Managers

### Outcomes
- Lower cost of change across frontend work
- Safer framework and dependency upgrades
- Fewer cross-domain surprises in delivery
- Better predictability for roadmap execution

### Measurability
- Weekly governance reports with trend signals
- Decision logs for traceability
- Baseline and progress snapshots for architectural health

## For Technical Leads

### Governance Support
- Boundary enforcement patterns for Angular + Nx
- CI rules for dependency direction and drift control
- Decision-log templates and review cadence
- Architectural health metrics and PR-scope analysis
- AI-assisted review workflows under explicit guardrails

### AI Stance
AI supports governance. It does not replace it.

Use AI for acceleration, not architectural authority.

## Scope Boundaries

### What We Do
- Stabilize codebases through governance-first structural work
- Improve ownership clarity and dependency safety
- Reduce regression risk with incremental, reversible changes

### What We Do Not Do
- Own feature delivery
- Run rewrites or big-bang migrations
- Act as generic staff augmentation
- Claim ownership of entire enterprise architecture bottlenecks

## Open Source vs Paid Value

### Public
- Patterns, templates, and examples in this repository
- Governance-oriented documentation and reference structures

### Paid
- Guidance and implementation support in production context
- Ongoing governance cadence and reporting
- Accountability for measurable structural progress

## Engagement Model
- One fixed day per week
- Remote collaboration
- Minimum three-month window
- Incremental and reversible changes

## Repository Role

This repository is a constraint system for calm, predictable frontend governance.

It demonstrates a DDD-inspired frontend modulith approach where:
- domain boundaries are explicit
- dependency direction is enforceable
- cross-domain changes are intentional
