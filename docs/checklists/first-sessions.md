## First Sessions

### Intent
Deliver low-risk refactors, set guardrails, and establish a sustainable cleanup cadence.
If cleanup happens one day per week, treat each "Day" below as one cleanup session.

### Session 1 | Triage and plan
- Confirm top 3 risks from the first 4 hours
- Map ownership and high-impact areas
- Pick 1-2 high-value, low-risk refactors

### Session 2-3 | Refactor safely
- Reduce 1 large component or god service
- Extract one reusable UI component
- Add tests for a critical path

### Session 4 | Guardrails
- Add lint rules or boundaries checks
- Establish naming and folder structure conventions
- Document patterns and anti-patterns

### Session 5 | Stabilize and report
- Verify tests and build status
- Summarize wins, remaining risks, and next steps
- Align on a 2-4 session roadmap

### Outputs
- 1-2 refactors merged
- Guardrails documented and enforced
- Sessions 2-4 cleanup plan
