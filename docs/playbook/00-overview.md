## Overview

Angular Cleanup focuses on improving **how Angular is used**, not forcing version upgrades, by reducing cognitive load, enforcing boundaries,
and replacing accidental complexity with boring, predictable patterns.

### What We Mean by Modern Angular

| Practice | Minimum Angular Version |
|--------|-------------------------|
| Standalone components | 14+ |
| Typed reactive forms | 14+ |
| Signals (optional) | 16+ |
| Explicit side effects | any |
| RxJS hygiene | any |
| Strict boundaries | any |
| Pragmatic testing | any |

Most improvements can be applied incrementally within an existing codebase.

Core principles:
- Standalone components only
- Signals-first state management
- Explicit side effects
- Strict dependency boundaries
