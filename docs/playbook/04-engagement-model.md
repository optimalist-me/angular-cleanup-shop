## Engagement Model

Typical engagements:
- 1-day cleanup audit
- Cleanup sprint (1–2 weeks)
- 1 day/week continuous hygiene

Deliverables:
- Findings report
- Refactor PRs
- Updated playbook & checklists

### Angular Upgrades

Angular version upgrades are treated as a **separate concern**.

We distinguish:
1. Architectural cleanup (default scope)
2. Version upgrade (optional, separately scoped)

This allows teams to:
- gain stability and clarity first
- reduce upgrade risk later
- avoid "big bang" migrations
