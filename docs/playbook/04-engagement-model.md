## Engagement Model

Typical engagements:
- 1-day cleanup audit
- Cleanup sprint (2–4 sessions; spans more than one sprint in calendar time)
- 1 day/week continuous hygiene

Note: Because work happens one day per week, onboarding and cleanup guidance is organized by sessions, not calendar weeks.

Deliverables:
- Findings report
- Refactor PRs
- Updated playbook & checklists

### Angular Upgrades

Angular version upgrades are treated as a **separate concern**.

We distinguish:
1. Architectural cleanup (default scope)
2. Version upgrade (optional, separately scoped)

This allows teams to:
- gain stability and clarity first
- reduce upgrade risk later
- avoid "big bang" migrations
