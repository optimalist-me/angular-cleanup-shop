Core principles

  • Standalone components and clear composition.
  • Signals-first state with computed derivations.
  • Strict dependency boundaries across domains.

Modern Angular means

  • Typed reactive forms and explicit side effects.
  • RxJS hygiene without nested operator chains.
  • Pragmatic testing and predictable change detection.

Upgrade stance

Upgrades are never assumed. Cleanup works within your current version and reduces future upgrade risk.

Version assessment

  • Current Angular version and supported practices.
  • Upgrade feasibility and risk, without assuming upgrades.
  • Constraints that impact cleanup sequencing.

Stabilize first

  • Identify entropy hotspots and regression risk.
  • Agree on boundaries and ownership early.
  • Define safe, incremental cleanup steps.

Component scope

  • Single responsibility and clear inputs/outputs.
  • Template complexity kept low.

State ownership

  • Local signals by default.
  • Computed state for derived values.

RxJS hygiene

  • Flattened operator chains.
  • Explicit side effects and cleanup.

Recommended metrics

  • Component LOC under 300, service LOC under 200.
  • No circular dependencies between domains.
  • Lazy-loaded feature routes where applicable.

Guardrails

  • Nx module boundaries and lint rules.
  • PR checklists that favor clarity over churn.
  • Small, reviewable refactor increments.
