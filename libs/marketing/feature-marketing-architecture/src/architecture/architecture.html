Frontend rule

  • UI uses data-access APIs only.
  • No server entities or internal microservice types.
  • DTOs (or schemas) define the boundary.

Why it helps

  • Backend can evolve without UI rewrites.
  • Testing stays fast and focused.
  • Contracts remain stable over time.

What it prevents

  • Shared becoming a dumping ground.
  • Cross-domain imports ‘just this once’.
  • Hidden dependency chains and coupling hubs.

What it enables

  • Clear public APIs per domain (what others may depend on).
  • Incremental refactors with predictable blast radius.
  • Safer upgrades later because structure is calm today.

Tooling (examples)

  • Module boundary enforcement (Nx or ESLint, depending on your stack).
  • CI checks for forbidden imports and circular dependencies.
  • Gradual constraint tightening: block new violations, reduce legacy ones.

AI is a tool — not a plan

  • AI can accelerate refactors, but only within clear boundaries.
  • No secrets, credentials, or customer data in prompts.
  • Sequencing and ownership stay human-led.

Where AI helps

  • Detecting boundary violations and circular dependencies.
  • Suggesting safe, incremental refactors.
  • Generating tests and documentation scaffolding.

Where governance leads

  • Defining domain ownership and public APIs.
  • Sequencing changes to minimize blast radius.
  • Deciding what should not be abstracted or shared.
